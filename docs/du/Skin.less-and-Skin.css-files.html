    Skin.less and Skin.css files

    Overview

    We have created Skin.css stylesheet for the default styling of web parts on your site. Using Skin.css helps to display most of the web parts correctly and ready for use. On the other side it might mildly interfere with your custom stylesheet. Skin.css is included in all our sample sites except the Blank site. There are several ways for how to use it in the cooperation with your custom site stylesheet:

    • Use !important directive or just stronger selector (with more classes) in your CSS.

    • Remove unwanted styles from Skin.less (this will probably affect more sites in your Kentico instance).

    • Do not include Skin.css on your site and just copy useful rules from it to the site stylesheet.

    Usage

    There is actually a global Skin.css file which is located in \App_Themes\Global\ and is included in the site Skin.css file. For example, our Corporate Site sample site has its own Skin.css file in \App_Themes\CorporateSite\ folder. If you open this file, you'll see that it just imports the global file. Taking a closer look, you can spot additional CSS declarations which just add specific styles just for this site and leave global Skin.css file untouched. This is the way we recommend using it on your own site. On the other hand, feel free to edit the global skin stylesheet, but keep in mind these customization when performing upgrade of your site. As another option, you can copy over just particular rules to your site stylesheet as mentioned above.

    Less code base

    Skin.css is actually generated from LESS Skin.less file. In fact, you could work with and modify Skin.css, but you would give away the opportunity to take advantage of structured, commented and more readable file (.less). If you are uncertain, how to work with this type of files, you can find articles well describing the process at Using the LESS CSS Preprocessor for Smarter Style Sheets.

    Structure and location

    Skin.less file is located in \App_Themes\Global\ folder and includes multiple skin sub-files in \App_Themes\Global\Skin\ folder.

    All styles are enwrapped in .ContentBody class which is there for a specific reason – to have more specific selectors over some UI elements. Obviously, this situation has a downside for being too specific over your site stylesheets. When such situation occurs, you should be able to pretty easily modify this Skin.less file.

    The file covers the following areas:

    • Pagination

    • Forms and filters

    • Shopping cart

    • Animations

    • Drop-down menu

    • Breadcrumbs

    • Alerts

    • Calendar

    • Tabs

    • UI Dialogs

    • Tables

    • Media library

    • Image editor

    • Document library

    • Categories

    • Context menu

    • Form controls (Slider, Spinner, Date time form control with calendar)

    • Button groups / Vertical separated buttons
