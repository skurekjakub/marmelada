    General hints

    Breadcrumbs

    Breadcrumbs in Admin / Semi admin UI web parts <Admin UI link> use UI markup. It means there is .breadcrumbs-dialog wrapper including .breadcrumb unordered list.

    However, Breadcrumbs web part still uses the same markup as in Kentico 7 - i.e. no wrapper (can be added via content before and after) listing links with .CMSBreadCrumbsLink class.

    Buttons

    In Kentico 8, all buttons have .btn class inherited from bootstrap. When combining .btn with other classes in stronger selectors, it may affect design of header actions buttons at Page and Design tab in the Pages application. There is a couple of ways for how to deal with this:

    • Do not use classes from page body (like .ContentBody or browser type classes) combined with .btn in a selector

    • Use header actions buttons wrapper class and rewrite changed properties for .cms-edit-menu .btn to default values

    Dropdown menu and toggle button

    When admin UI context menu (it can be displayed by right-clicking the group in the Groups viewer web part) looks bad because of your site CSS, try to use this code to fix it up: .dropdown-menu li { margin: 0 !important; padding: 0 !important; list-style: none !important; }

    If you would like to modify the background color of active dropdown-toggle button, use .open .btn.dropdown-toggle selector with !important directive or .btn-actions .btn-group.open .btn.icon-only.dropdown-toggle selector.

    For styling borders between the buttons in .btn-group, use .btn-group > .btn:not(:last-child):not(.dropdown-toggle) { border-right: 1px solid #YOURCOLOR; }
    Note that this is considered as semi-admin UI (not supported in IE<9).

    For fixing height in dropdown toggle button in .btn-actions set .dropdown-toggle { line-height: normal; }

    Error labels

    For styling error labels displayed on your live site, use one of these CSS selectors: .ContentBody .form-control-error, .ContentBody .ErrorLabel, .ContentBody .LineErrorLabel or .ContentBody .EditingFormErrorLabel

    Filters

    Advanced filter structure is now based on the forms markup (see Forms <Forms link>). The .form-horizontal wrapper has an additional class .form-filter.

    Forms

    Default layout of the forms has been completely redesigned and inspired by bootstrap. Now, most forms use <div> based layout. The whole forms are wrapped by .form-horizontal class, the rows of the forms (containing title label, input and validation label) are represented by .form-group class. Title label is wrapped by .editing-form-label-cell class and input (which can be, for example, textbox, dropdown or radio button list) potentially with validation label is wrapped by .editing-form-value-cell class.

    <div class="form-horizontal">
    <div class="form-group">
    <div class="editing-form-label-cell">
    Field label
    </div>
    <div class="editing-form-value-cell">
    Field input
    </div>
     </div>
    <div class="form-group">
    <div class="editing-form-label-cell">
    Another field label
    </div>
    <div class="editing-form-value-cell">
    Another field input
    </div>
    </div>
    </div>

    If you notice a big space above the form when using Skin.css, it might be caused by clear property which is set for .form-group:after by .clearfix() mixin in Skin.less. Adding the .form-horizontal .form-group:after { clear:none; } rule to your CSS should solve the issue.

    Lorem ipsum macro

    Rating stars

    If you are using rating stars, you should use these selectors for styling their sizes:

    • .rating-star { font-size: 8px !important; /* this is the space between stars */ }

    • .rating-star:before { font-size: 16px; /* this is the star */ }

    Search box

    Search box styling is now based on .form-horizontal class (see <Forms>). In Skin.css, it has default width set to 660px. If this causes a problem with displaying the search box, we recommend one of following solutions:

    • Remove the rule .form-horizontal { width: @screen-tablet; } from Skin.less (it will influence all sites on the instance)

    • Use stronger selector (at least 3 classes or !important directive) for overwriting .ContentBody .form-horizontal { width: 660px; } rule in Skin.css (for example, wrap the search box with a .searchTextMode class and set the width for .ContentBody .searchTextMode .form-horizontal)

    Tables

    Tables can be found all over the UI of Kentico 8 as well as on couple of places on the live site - e.g. My Account (Subscriptions, Addresses, Messages), Projects, etc. You may know them as UniGrids. However, we started to call them tables because, well, they are tables in the final output (CSS classes naming remains the same).

    Table is enwrapped by .unigrid and .unigrid-content classes. On table itself are used .table .table-hover clases. Table header row is represented by .unigrid-head class in thead. Speaking about other rows, there are classes for particular type of content e.g.: .ArrowIcon, .FileTypeIcon, .RowContent.

    Table actions

    There is definitely a selection option if, for example, the table contains list of user messages to be selected. They use class .unigrid-selection with a chec-kbox inside.

    There can also be actions like delete, view, reply etc., which are inside the .unigrid-actions class. In case there are additional actions in the context menu, the .unigrid-actionmenu wrapper class is used.

    Context menu

    If available, the context menu is located in div below the table enwrapped by .ContextMenu class which is the same as in the previous version of Kentico. However, the context menu also has the .cms-bootstrap class . Each item in list sits in .ItemPadding wrapper and is simply in span element with the .Name class and image removed.

    Pagination

    Pagination contains the following blocks: List, Pages and Items per page.

    List is an unordered list with class .pagination-list. Active page link is marked with .active class.

    Pages contains a select enwrapped by .pagination-pages class.

    Last block is Items per page select which is in .pagination-items-per-page class.

    Mass action

    Some tables have an ability to perform a mass action over the listed items. They are offered in a form-like markup: wrapped into .form-horizontal .mass-action classes. Each row has its .form-group and selects and button are inserted in .mass-action-value-cell wrapper.

    Tabs

    There are currently two types of tab headers and tab content wrappers: .TabContainer + .TabBody and .TabsHeader + .TabsContent.

    .TabBody or .TabsContent contains content of currently displayed tab, meanwhile .TabContainer or .TabsHeader contains the list with names of all tabs to select.

    In some cases, the old markup of tabs names has been replaced by bootstrap based structure used in the administration UI.

    <div class="nav-tabs-container-horizontal">
    <ul class="nav nav-tabs">
    <li class="active"><span class="tab-title">Inbox</span></a></li>
    <li><a class="Link" href="..." target="_self"><span class="tab-title">Outbox</span></a></li>
    <li><a class="Link" href="..." target="_self"><span class="tab-title">Contact list</span></a></li>
    </ul>
    </div>

    Tree view

    Tree view control used for example for Tree view web part or Categories tree, has been slightly changed. The main difference in Kentico 8 is that it uses font icons instead of images. It means there is an <i> element preceding a link element. This i element has basically a class, which determines which font-icon will be used, like this: .icon-doc. That's why we load core-icons font for this purpose from within \App_Themes\Default\Fonts\ folder. You can see more details about font icons here: <FONT ICONS LINK>.
