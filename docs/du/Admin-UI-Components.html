    Admin UI Components

    Styling

    • Component base styling in Skin.less.

    • Site specific styles in Site stylesheet.

    • Buttons do not have fixed width.

    • Select input cannot be styled multiplatform way in IE7 for Non admin UI.

    • Check-boxes in forms have their label on the right side in the same value cell (preferably).

    • Icons on live site can be both - images and font-icons. However we prefer font-icons or nothing (new buttons style).

    • Bootstrap stylesheets are linked in "editor" interfaces - i.e. User contribution dialogs, Dialogs with (uni)selector.

    • Blank Site and Blank Site ASPX don't contain skin.css.

    • Intranet Portal still has additional skins (applied on selected Departments and Workgroups), Personal Site is cut off - no skins anymore.

    Admin UI components

    • Uses UI styling and look and feel

    • Browser support same as in the UI
