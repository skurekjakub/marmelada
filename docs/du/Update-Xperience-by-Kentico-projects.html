    Update Xperience by Kentico projects

    Updates for Xperience by Kentico are released in the form of:

    • Hotfixes – weekly fixes of bugs and potential security issues.

    • Refreshes – more extensive updates that add new features, introduce changes, and fix issues. Note that refreshes may contain breaking changes to the code base. Before updating, check the Breaking changes section in Changelog to see if your codebase is affected. Refreshes are planned for every month, but the release schedule may vary depending on the features included in each refresh.

    We recommend that you update your Xperience projects regularly.

    Use the following process to apply updates:

    Before proceeding with the update:

    • Create a production database and project backup.

    • Stop your Xperience application, if it is receiving traffic.

    1. Set up a local development project, including the database. You can either maintain a permanent local development instance or backup and locally deploy the production project and database for each update.

    2. Update your local project's Xperience NuGet packages.

    3. Update your local database and content files.

    4. Test that the updated local project works correctly.

    5. Deploy the local project files to your production environment (using your preferred publishing process).

    6. Apply the database update to your production database.

      • This ensures minimum downtime for your live site while avoiding loss of data.

      • For example, you can switch to a separate "live" configuration of your local project that connects to the production database, and repeat the update of database and content files.

    In addition to updating individual projects, we also recommend updating the templates used for installing new projects. See Update project templates.

    Check the Changelog page for additional details, workarounds and instructions related to individual update versions.

    Update NuGet packages

    Start by updating your local project's Xperience code libraries and administration, which are provided in the form of NuGet packages.

    1. Open your application in Visual Studio.

    2. Right-click your website project in the Solution Explorer and select Manage NuGet Packages.

    3. On the Updates card, select all Kentico.Xperience NuGet packages and click Update.

      • Kentico.Xperience.WebApp

      • Kentico.Xperience.Admin

      • All installed extension packages

    4. Rebuild the solution.

    The application will not start until the NuGet package version matches your database version. Continue by updating your local database and content files.

    Update database and content files

    After updating your local project's NuGet packages, use the Xperience .NET command line utility to update the local database and content files.

    To update the database and content files:

    1. In the command line prompt, navigate to the root folder of the project you want to update.

    2. Run the following command:

      dotnet run --no-build -- --kxp-update

      Command parameters

      Add the --no-build parameter to the dotnet run command if you have already rebuilt your project after updating the NuGet packages. This option saves time and avoids another unnecessary project build before the database update.

      Add the -- syntax separator before any command parameters that you wish to pass directly to the Xperience application (--kxp-update). This prevents conflicts with general .NET CLI parameters, for example when using the --help option.

    Your local database and content files are now updated, and the database version should match the version of the project's Xperience NuGet packages.

    The database update process does not automatically start the application when finished. You can restart the application using the dotnet run command, or another suitable approach depending on your hosting environment.

    Updating development projects with Continuous Integration

    If you are updating a development project with Continuous Integration enabled:

    • Before running the database update, we recommend disabling Continuous Integration (use the settings in the Xperience administration).

    • Apply the update separately for each instance in your development environment. You cannot fully synchronize the database update using your source control system and Continuous Integration.

    • After updating, every developer needs to run complete serialization for all objects to recreate the content of the CIRepository folder on their instance (use the --kxp-ci-store CLI command).

    • After one developer commits the updated changes to the source control, other developers CANNOT commit or load changes until they apply the update to their own instance.

    After updating

    All users who work with the Xperience administration should clear the cache in their browsers after the update. Otherwise, some features in the administration interface may not be displayed correctly.

    Update project templates

    If you have installed the Xperience project templates package on your machine, we recommend keeping the templates up-to-date. This is required if you wish to create new projects that include released features and fixes.

    To update the Xperience project templates, open the command line and run the following command (depending on the .NET SDK version installed on your machine):

    .NET 6 SDK
    dotnet new --update-apply
    .NET 7 SDK
    dotnet new update
